Introduction

jflatdb is a lightweight, Python-based flat-file database framework designed for fast, simple, and file-based data storage without the need for a server or SQL engine. It's ideal for developers building CLI tools, microservices, or offline applications that need persistent storage without the overhead of traditional database systems.

Built entirely in Python, jflatdb provides a familiar, SQL-like interface for querying JSON data stored in flat files. It offers features like indexing, caching, transactions, and schema migrations‚Äîall while maintaining simplicity and ease of use.

Installation

From PyPI

Install the latest stable version using pip:

pip install jflatdb

From Source

Clone the repository and install locally:

git clone https://github.com/jflatdb/jflatdb.git
cd jflatdb
pip install .

Requirements

jflatdb requires Python 3.7+ and has no external dependencies.

Quick Start

Basic Usage

from jflatdb.database import Database

# Initialize database
db = Database("users.json", password="your-password")

# Insert data
db.insert({"name": "Akki", "email": "akki@example.com", "age": 25})
db.insert({"name": "Bob", "email": "bob@example.com", "age": 30})

# Find records
users = db.find({"name": "Akki"})
print(users)  # [{"name": "Akki", "email": "akki@example.com", "age": 25}]

# Update records
db.update({"name": "Akki"}, {"email": "new@email.com"})

# Delete records
db.delete({"name": "Akki"})

Advanced Queries

# Greater than / Less than
young_users = db.find({"age": {"$gt": 18, "$lt": 30}})

# IN operator
specific_ids = db.find({"id": {"$in": [1, 2, 3]}})

# LIKE for pattern matching
names_starting_with_a = db.find({"name": {"$like": "A%"}})

# BETWEEN
ages_in_range = db.find({"age": {"$between": [20, 30]}})

How to Contribute

  1. Fork the repository on GitHub
  2. Clone your fork: 
    git clone https://github.com/YOUR_USERNAME/jflatdb.git
cd jflatdb
  3. Create a new branch: 
    git checkout -b feature/your-feature-name
  4. Make your changes and test thoroughly
  5. Run tests: 
    pytest
  6. Commit your changes: 
    git commit -m 'feat: add new feature'
  7. Push to your fork: 
    git push origin feature/your-feature-name
  8. Open a Pull Request with a clear description

Commit Message Guidelines

We follow Conventional Commits format:
